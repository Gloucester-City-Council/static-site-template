# Data Classification – GitHub Pilot

This organisation is used by the Council for a **pilot of publicly accessible, static websites and supporting documentation**.

All repositories in this organisation are assumed to be **PUBLIC by default**.

---

## Permitted information

The following types of information **may be stored** in GitHub repositories within this organisation:

- Public website content:
  - HTML, CSS, JavaScript
  - Images, icons, fonts
- Open documentation and guidance
- Open-source code and libraries
- Public policy summaries and explanatory content
- Design assets intended for public release
- **Publicly available details of Councillors**, where the information is already published by the Council or required by statute
- **Publicly available details of senior managers**, where such information is already in the public domain (e.g. senior leadership team pages)

All content must be suitable for:
- Publication on the Council’s public website
- Freedom of Information (FOI) release
- Public and media scrutiny

---

## Prohibited information

The following **must not be stored** in any GitHub repository in this organisation:

### Personal data (non-public)
- Personal data relating to identifiable individuals that is **not already lawfully in the public domain**, including:
  - Residents
  - Staff below senior management level
  - Private contact details for any individual
- Internal staff lists, distribution lists, or directories
- User-submitted content
- Tracking or analytics data capable of identifying individuals (e.g. full IP address logs)

### Special category or sensitive data
- Health information
- Safeguarding data
- Criminal records or allegations
- Equality monitoring data
- Any data protected under UK GDPR special category definitions

### Security and operational data
- Passwords, API keys, tokens, certificates
- Connection strings or authentication secrets
- Internal system URLs, IP addresses or hostnames
- Network diagrams or infrastructure configuration
- Backup files or system exports

### Commercial and legal material
- Contractual documents not approved for publication
- Licensed content without explicit permission for public reuse
- Proprietary supplier materials

---

## Information classification alignment

For the purposes of UK local government information governance, GitHub pilot content is restricted to:

✅ **OFFICIAL – PUBLIC**

The following classifications are **explicitly excluded**:

❌ OFFICIAL – SENSITIVE  
❌ Any Protective Marking or Security Classification  
❌ Any pre-publication draft requiring controlled circulation  

If content cannot safely be classified as **OFFICIAL – PUBLIC**, it must **not** be placed in GitHub.

---

## Responsibility for classification

- Repository owners are responsible for ensuring content complies with this classification.
- Contributors must check content before committing it.
- Organisation Owners reserve the right to remove content that breaches this policy.

If you are unsure whether information is appropriate for GitHub:
- Do **not** upload it.
- Seek advice from the Digital team, Information Governance Team or the Data Protection Officer.

---

## Breach handling

If prohibited information is identified in a repository:

1. The content must be removed immediately.
2. The Organisation Owners must be notified.
3. The Data Protection Officer must be informed via:
   **dataprotection@gloucester.gov.uk**
4. Git history may be rewritten where necessary to fully remove the content.

---

## Review and changes

This data classification applies for the duration of the GitHub pilot.

It will be reviewed:
- If the scope of GitHub usage changes
- If integration with live systems is proposed
- As part of any transition to GitHub Enterprise
